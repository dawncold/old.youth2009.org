是不是技术人员都喜欢买书？

Jul 5th, 2011 

        <p>自认为我也算是一个追求技术的人了，现在更加喜欢技术的世界了，那里没有一些弹性的东西，不像官场那样可以商量，严格地执行着计算机的语言——0、1。</p>

作为一个技术人员，我发现自己很喜欢买书，当然买的最多的可能就是技术方面的书籍，包括一些被称为计算机编程名著一类的书，当然这些书太多了，在网上混迹时间一长，经不住别人的鼓动、诱惑，终于也走上了收集名著的行列之中，买书之前总是希望自己能够通过这本书学到什么高深的技术，在职场、创业中屹立于不败之地。但买回来之后，并不像我期待的那样，终归还是有些好处的，只是要打些折扣了：）但总归还是比不看书荒废了那段时间好吧。

除了买一些技术性的书籍之外，我还喜欢买一些哲学、心理学、散文等方面的书来看，这和个人的爱好有点关系，但哲学、心理学这类书买得很少，自己只是停留在喜欢的角度，不是非得去研究，所以就不太深入。

不知怎么我又喜欢上了经济学的东西，原因很复杂，简单地说是事物之间那种千丝万缕的联系，而且我发现计算机系和经济系有着无法分割的联系，所以我还是喜欢点经济吧。

今天稍微有点冲动买下了张小娴的一本新作——《想念》。首先声明一下，我不是因为这个名字才去买的，是我本身就喜欢张小娴写的一些文章，虽然自己没看过几篇，但有些耳熟能详的还是听过，比如“世上最远的距离不是⋯⋯而是⋯⋯”。一个计算机人士光是看技术书籍只能提高技术能力，而光靠技术，在这个社会中还是很难过上好日子的，如果你只是希望饿不着那就算啦。

Python中print中的逗号

Jul 3rd, 2011 

        <p>以往可能会写这样的代码：</p>


print x 



print y


 当然，x、y在前面都是定义过的变量，此时，如果在x后面加一个逗号，再看输出结果，print就不会再换行了，只会在输出完x后加一个空格接着就输出y了，这点不错！

万事唯有坚持

        <p>本以为这个话题我以前似乎说到过，翻了一会日志却没找到。</p>

这是刚刚看Mac下的FIT输入法主页的时候想起来的，主页上有个广告：“5年20万果粉的选择”。20W这个用户量相当大了呢，暂且不说有没有统计问题，这么大的用户量绝非一日之功，再看前面，5年啊！输入法是一个基础性的软件，每天都会用很多次，坚持开发维护了5年的这样一个软件，不断积累了这么庞大的用户，的确是不一般。真想采访一下他们在这5年来的所感所想。

iphone上的应用，愤怒的小鸟，也是在做了很多产品之后继续坚持下去才获得今天的成功；植物大战僵尸的公司，在制作这款游戏时竟然优化了3年，这等坚持，不是一般的！

还有恋爱过程中的各种坚持…

        <p>但愿这是最后一次讨厌的考试。</p>

原来上高中、初中的时候都是在盼望考试的时候能考个高分，而现在却变成对考试能够及格的盼望。其中落差还是挺大的吧。似乎随着我们上学的层次不断提高，我们对考试的要求越来越低了，这不就是我们教育制度的问题么？！虽然如此，我还是严格要求自己，可惜只要不是考试前夕，总也拿不出时间来复习，往往是考试的这几天很疲劳，考过之后成绩还算不错，于是下一年仍然这样准备。

下半年不知道还有没有时间考考软件设计师的证，虽然这个证在企业中用处不大，单至少是本专业国内来说比较高级的认证了，能够拿到总归是要付出一点努力的，我想对于经过大学学习的人来说，努力拿证问题不大，最大的问题可能就是找不到那么多时间来做。真想把自己的空闲时间积累起来一块用啊，可惜不行，似乎时间这种资源是唯一的，不能你想要积累就积累起来，如果允许你积累时间，那么相应地别人的那块时间就必须压缩掉，这样反而很不公平了。或许下个世纪能够突破时间吧！

针对github的两条设置

Jun 21st, 2011 

        <p>刚刚看到github的help页面上有两条关于git的设置，是针对github用的：</p>


git config --global github.user USERNAME 



git config --global github.token TOKEN


 其中的TOKEN要从github网站的帐户管理中得到，而且这个TOKEN和密码相关，如果你更改了密码，需要重新设置。但我还不清楚设置了这个对于github的提交有什么特殊的好处，等以后用用再说吧。
