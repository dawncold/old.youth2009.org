信用卡注册流程

Feb 26th, 2011 

        <p>我指的银行是中国工商银行，其他的银行可能有些差别。</p>

首先是申请相关信用卡业务

拿到卡后相关工作人员会提示你启用这张卡

如果你以前拥有普通的银行卡可以考虑把信用卡和以前的这张卡绑定在一起，因为我以前有一张银行卡并且开通了网银，有了信用卡之后就从网银的界面中绑定了信用卡，这样你在查看帐户信息的时候会显示信用卡是“网银注册”，而原来的卡会显示“柜面注册”，这个“网银注册”的权限是不高的，比如信用卡的“境外无卡支付”就不能自助开通，需要打电话。最要命的是这样的信用卡不支持网银功能！也许网上转帐都不行！！

遇到了我这样的情况，请先在网银中删除自己绑定的信用卡，然后去附近的银行营业网点办理柜面注册业务。

我的目标

Feb 25th, 2011 

        <p>人类如果缺失了目标，那么在社会上活着确实是盲目的；假如长期缺乏目标，那就很可悲了，因为活了好久却不知道是为了什么活着，还不如死掉一了百了。当然我说得可能有些激进，可现实确实如此！</p>

今天我用了很长时间思考我的目标是什么，不止是今天，很长一段时间就这样思考过了，我还买了程序员职业规划方面的书来学习，可惜那本书根本没有帮忙分析怎么规划。我手头边有很多很多的书，而且每一本都很厚，甚至每一本都能作为一个独立的科目来研究下去。曾想安安稳稳地朝着一个方向发展，现在的“战线”太长了，而个人能力有限，这样的结果想必是很明了的──样样不精。

原本的我对于样样不精这种现象是极力规避的，我不想成为那样的人，可这个时代驱使着我要那样做！我已经离那个原点太远了，可能根本回不去了，甚至说这个时代的人们都必须朝着另外的方向发展！

以上的这些变化让我想到了尼采的断言──“重估一切价值”。

现在的社会价值观一点点侵蚀着我们每个人，而且这已经到了需要我们重新评估一切价值的时候。这不是一场实实在在的战争，但影响力绝对不亚于真实的战争，这就是思想的力量。

那么，我的目标呢？

几乎每个寒暑假我都会尝试一个新的领域，这种好奇不也应该是一个人该有的品质么？然而我就是被这些新的技术一点点瓜分掉了精力，不知道该朝着哪个方向走，还总想找到一个冠冕堂皇的理由支配自己一定要选出来几个努力研究下去！技术上的各路大牛们还都能相信“让自己快乐的就是最适合自己的”这样的观点，然而我现在根本找不到那所谓的快乐所在，在我一贯称为喜欢的计算机行业里，我找不到能让我高兴的地方，这可能就是我困惑的原因。

同样在感情方面我也遇到了不少挫折，一直喜欢着一个现在还不怎么喜欢我的人，而且我一直不断地在努力，长久的没有反馈似乎已经影响到了我的人生目标，我想情感方面的失败对我目标缺失的影响可能更大一些。

目标？继续寻找！

cd gitosis-admin
vi gitosis.conf



增加新的项目名字和相关配置

Git的学习笔记

Feb 23rd, 2011 

        <p>1.从http://code.google.com/p/git-osx-installer/downloads/list下载最新的git安装包</p>

2.安装

3.配置git


git config --global user.name 'dawncold'



git config --global user.email loooseleaves@gmail.com


4.学习git 建立一个项目文件夹


mkdir studygit

进入这个文件夹

cd studygit/

初始化git

git init

返回这样一句:

Initialized empty Git repository in /Users/dawncold/Documents/Project/studygit/.git/

然后创建几个文件来试试git的跟踪:

vi test.c



vi README


加入git跟踪:

git add *.c



git add README


提交一下

git commit -m 'init project----studygit'

返回了:

[master (root-commit) 2e2368b] init project----studygit



2 files changed, 6 insertions(+), 0 deletions(-)
create mode 100644 README
create mode 100644 test.c


查看一下git状态

git status

返回:

# On branch master



nothing to commit (working directory clean)


表示现在跟踪的这些文件没有更改，如果更改了就会显示出来有差异。

如果在这个目录下建立了新的文件，再次查看status会出现一个untracked files:，此时要想加入到git跟踪，用git add xxx即可。

 

git有个看起来麻烦的地方:如果add了a这个文件，然后再更改一下这个文件，比如加了一句话，此时查看status会发现untracked中又出现了刚才的那个a文件，如果此时commit的话我们刚刚改的a文件不会被提交上去，只会提交那个add后而且没有再更改的a文件。此时应该再add一次才可以!!!(用git commit -a即可略过add的步骤，只要跟踪过a这个文件，那就肯定会提交最新的)

 

git的忽略文件，在本项目的文件夹中建立一个叫.gitignore的文件，里面加入要忽略的文件名字，似乎能用正则表达式，比如:

*.[oa] 是忽略o和a结尾的文件;*~ 是忽略带波浪线的文件，这类文件一般是文本编辑器创建的备份。要养成良好的习惯，项目一开始就建立这个文件。

 

git的删除文件，如果用git rm xxx则在版本仓库中删除，并且在目录下也删除了，如果用git rm –cache xxx则只是在版本仓库里删除跟踪。

 

git的重命名文件，一般在linux下都是使用mv命令，在git中也是如此，git mv xxx，git可以了解你做了重命名还是其他的。

 

git的撤销，刚刚提交的文件中少了一个，可以这样做:

$ git commit -m 'initial commit'



$ git add forgotten_file
$ git commit --amend


后续内容请查看progit这本书在github上的翻译，我就是对着翻译来写了这一小篇文正，还有不少遗漏的地方没有写出来。
